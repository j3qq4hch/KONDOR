# Kondor

Кондор это как орел, только летает выше

## Зачем это все

Kondor - это ПО для проектирования электронных устройство. Это существенно отличает его от существующего ПО, вроде Eagle или Kicad. Последние являются ПО для проектирования печатных плат. Однако, печатная плата почти никогда не имеет смысла сама по себе: по крайней мере она располагается в корпусе. А в реальных устройствах часто бывает несколько печатных плат, связаных друг с другом разными типами разъмов, которые располагаются в одном общем или нескольких корпусах. Именно для описания этой совокупности печатных плат, разъемов и механических частей и нужен Kondor. 

На данном этапе Kondor не является ПО для разработки печатных плат, в нем нет рендера, он не управляет компонентами. Для просмотра топологии плат Kondor использует EagleCAD V7.7 или KiCad. Сам же Kondor оперирует 3D моделями печатных плат(если такие есть), моделями механических частей (корпусами и кронштейнами) и информацией о связи печатных плат между собой через разъемы. 

## Визуальное представление

Главное окно Kondor - 3D-viewport, в котором отображаются все основные части устройства в виде 3D моделей или временных заглушек-коробок, если 3D модели нет. В левой части экрана отображается вертикальный список всех сущностей в устройстве, сгруппированный по типу: платы или механические части. При клике на 3д модель сущности ее имя подсвечивается в списке. При клике на имя сущности в списке, вокруг модели возникает индикатор выбора. При двойном клике на 3д модель сущность или на ее имя в списке открывается окно свойств. 

### Электрические связи плат между собой

У каждой платы есть разъемы. Разъемы на одной плате, очевидно, должны быть электрически и механически совместимы с разъемами на плате, с которой они соединияются. Для пометки объекта на плате в качестве разъема, который Kondor должен учитывать, требуется добавить к нему атрибут с именем ConID и уникальным значением, которое должно совпадать с ConID разъема на другой плате, с которым данный разъем должен соединиться. Кроме этого разъем может иметь атрибут ConType, принимающий два значения: b2b или w2b. Если этого атрибута нет, то разъем считается w2b.

Разъемы бывают двух типов: board-to-board (b2b) и wire-to-board. 

* board-to-board разъемы служат для жесткого соединения двух плат. Классический пример - PLS/PBS разъемы и мезонинная установка печатных плат (ардуино шилд). Ключевая особенность такого соединения - распиновка разъема на обеих соединяемых платах должна быть одинаковой. В таком соединении всегда только 2 участника.
Также важна механическая совместимость, особенно если межплатных разъемов более одного. 

* wire-to-board разъемы служат для подключения, очевидно, проводов к плате. вообще говоря, одинаковая распиновка для таких разъемов необязательна, т.к. на уровне кабельной сборки возможны вариации распиновки. Также требования к механическому размещению этих разъемов несколько менее жесткие. Однако, w2b соединения бывают двух видов: соединение точка-точка (например, jst-xh разъемы) и соединения типа шина, когда к одному кабельному шлефу может быть подключено сразу несколько печатных платы (например, разъемы для IDC-шлейфов). 

Отобразить информацию о распиновке в 3D сцене, очевидно, затруднительно и излишне. Поэтому в 3D сцене должны отображаться только тонкие виртуальные линии, соединяющие соответствующие разъемы.  При двойном клике на такую линию должно открыватья окно описания соединения. В этом окне должны быть схематично изображены все сущности, объединяемые этой электрической линией. При этом распиновка самих разъемов может по тем или иным причинам не совпадать. В таком случае нужно выдавать предупрежденающее окно.

## Контекстное меню

Также можно кликать правой кнопкой мыши по имени или по 3д модели, это открывает контекстное меню, в котором есть следующие пункты:
1) Удалить - удаляет сущность из устройства. 
2) Свойства - открывает окно свойств
3) Заблокировать позицию - запрещает перемещение модели. Около имени сущности должна появиться иконка якоря.
4) Сбросить позицию - сбрасывает позицию сущности в окне 3д просмотра на дефолтную. Это может быть нужно, если пользователь "потерял" модель в окне 3д просмотра
5) Показать - перемещает камеру так, чтобы выбранная сущность оказалась в центре и занимала по крайней мере 2/3 экрана
6) Обновить с диска - заново запускает процедуру генерации 3д модели печатной платы или тесселяции степ-файла, или перегружает glb файл.

Все 3д модели можно двигать при помощи гизмо. Положение каждой модели запоминается после каждого передвижения, чтобы оно сохранялось при выключении программы или при перезагрузке модели.

## Добавление сущностей

Механические части добавляются через импорт step или gltf-модели. При импорте step модели она тесселируется и дальше используется уже в таком виде.
Импорт печатной платы из Eagle происходит при помощи brd-файла. При импорте Kondor создает 3D-модель платы с помощью NoABS.Eagle3d. Чтобы это работало, на ПК должен быть установлен Eagle 7.7, NoABS.Eagle3d и были в наличии актуальные библиотеки с 3д моделями. Например, NoABS.Libs. Если glb файл с именем платы уже существует рядом с brd файлом, будет использован он.
Импорт печатной платы из KiCad происходит при помощи kicad_pcb файла. При импорте Kondor создает 3D-модель платы средствами KiCad. Если glb файл с именем платы уже существует рядом с brd файлом, будет использован он.

## Отслеживание изменений

При импорте той или иной сущности Kondor запоминает дату последнего изменения файла. Во время работы Kondor периодически проверяет (или получает уведомления от ОС, если это возможно), не изменились ли исходные файлы. Если какой-то файл изменился, то рядом с именем сущности, с которой ассоциирован файл, должна появитья желтая иконка ВНИМАНИЕ. В таком случае пользователь можно обновить модель через контекстное меню по ПКМ. После обновления иконка должна пропасть.
